**PHASE 6 — APPLICATION MOBILE EXPO + INTÉGRATION API**



Objectif général :

Construire une première application mobile Expo / React Native robuste, typée, sécurisée et connectée au backend existant, sans modifier le comportement clinique, sans introduire de données réelles, sans automatiser de décision, et sans contourner les contrats API validés en Phase 5.



La Phase 6 doit créer l’interface mobile patient/clinicien du prototype, en consommant exclusivement l’API backend existante.



**Contraintes non négociables Phase 6 :**



1\. Données synthétiques uniquement

\- L’application doit afficher clairement que toutes les données sont simulées.

\- Aucun flux de données réelles.

\- Aucun capteur réel obligatoire.

\- Aucun appairage IoMT réel non maîtrisé.



2\. Open-loop strict

\- Aucune décision automatique.

\- Aucune modification de traitement.

\- Aucune dose.

\- Aucune instruction thérapeutique.

\- Les recommandations restent des suggestions soumises à validation clinicien.



3\. Sécurité mobile

\- Aucun token dans AsyncStorage.

\- Aucun token dans localStorage.

\- Utiliser SecureStore / Keychain / Keystore pour les tokens.

\- Ne jamais logger access token, refresh token, mot de passe ou données sensibles.

\- Logout doit effacer les secrets.

\- Refresh token flow propre.

\- Gestion expiration access token.

\- Pas de secret hardcodé.



4\. Respect strict des rôles

\- Patient :

&#x20; - accès uniquement à son dossier ;

&#x20; - lecture de ses données ;

&#x20; - lecture des recommandations approuvées uniquement ;

&#x20; - pas de génération de recommandation ;

&#x20; - pas d’approbation/rejet.

\- Clinicien/admin :

&#x20; - liste patients ;

&#x20; - lecture dossier patient ;

&#x20; - génération de suggestions ;

&#x20; - approbation/rejet/modification ;

&#x20; - accès XAI global/local selon API.

\- Non authentifié :

&#x20; - accès uniquement login/register si activé.



5\. Pas de changement ML/XAI/recommandation

\- Ne pas réentraîner.

\- Ne pas modifier les seuils.

\- Ne pas modifier les modèles.

\- Ne pas modifier la logique de recommandations sauf correction nécessaire documentée.

\- Ne pas réinterpréter la XAI comme justification clinique.



6\. API source-of-truth

\- L’app mobile ne calcule pas de risque localement.

\- L’app mobile ne génère pas de recommandations localement.

\- L’app mobile affiche les résultats fournis par l’API.

\- Les probabilités, XAI et recommandations viennent du backend.



7\. Disclaimers visibles

Afficher clairement :

\- prototype académique ;

\- données simulées ;

\- non certifié ;

\- ne remplace pas un avis médical ;

\- ne modifiez jamais votre traitement sans avis médical ;

\- XAI décrit le modèle, pas une cause médicale.
